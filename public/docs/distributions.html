OS Setup — Linux, macOS & Windows

CodeSleuth lives in a single directory on any OS. There are no separate distributions. The pipeline logic, agents, scripts, and documentation are identical across Linux, macOS, and Windows. Only a small set of OS-specific helper files differ — and they all ship in the same scripts/ folder.

All Python scripts (scripts/*.py) use Path(__file__).resolve().parent.parent to locate themselves — they are fully OS-agnostic. The Go script (parallel_runner.go) compiles natively on all platforms. All agent rules, catalogs, templates, and documentation are the same file on every OS.

Path Personalisation (first-time setup)

The pipeline ships with hardcoded paths pointing to /home/user/codesleuth/. If you install it at a different location or under a different username, run the setup script once to replace all path references throughout the pipeline files automatically.

Linux & macOS

bash SETUP.sh

Detects your actual install directory and replaces /home/user/codesleuth/ with it. Works on both Linux (sed -i) and macOS (sed -i '') automatically.

Windows

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\SETUP.ps1

Replaces /home/user/codesleuth with the actual install path using forward slashes (C:/Users/yourname/codesleuth), which Python, PowerShell, and Go all accept on Windows. If python3 is not found on PATH, the script also updates script invocations to use python.

Python Command

On Linux and macOS the pipeline calls python3. On Windows, if only python (not python3) is on your PATH, run SETUP.ps1 — it detects this and updates all invocations in the agent files automatically.
