💰 Token Tracking

Token Budget & Model Tracking

!budget shows what the pipeline can actually measure about session cost — model pricing, task activity, and exact token counts when an external harness supplies them. For real-time token totals, use Claude Code's built-in /cost command.

⚠️
Why token counts aren't always available

LLM agents cannot read their own token counts from inside a response — API usage metadata is not exposed through Claude Code's tool interface. session_tokens.jsonl is only populated when an external harness (a Claude Code hook, CI runner, or wrapper script) supplies real API counts. In most interactive sessions this file will be empty, and !budget will say so explicitly rather than guessing.

The !budget Command

Type !budget (or /budget) at any point. The command runs token_tracker.py budget and displays its output verbatim — no estimates, no freestyle.

Getting Real Token Counts

For exact token counts and cost in an interactive session, use Claude Code's native tracking:

  • During a session: type /cost in Claude Code — shows cumulative input tokens, output tokens, and total cost for the current session.
  • After a session: check the Anthropic Console → Usage dashboard for per-API-key spending.

How Token Data Is Recorded (external harness)

When an external harness (hook or CI runner) has real token counts, it calls token_tracker.py record:

python /home/user/codesleuth/scripts/token_tracker.py record \
    --agent "Builder" \
    --task "TASK-003" \
    --model "claude-sonnet-4-6" \
    --input-tokens 4821 \
    --output-tokens 1203 \
    --cache-read-tokens 18420 \
    --cache-write-tokens 800 \
    --log artifacts/telemetry/session_tokens.jsonl

Token counts must come from the API response metadata — available to the harness that made the API call, not to the LLM inside the response. A Claude Code hook listening to the Stop event (or a wrapper script around the Claude Code CLI) can supply these counts and call record.

Storage

Records are appended (newline-delimited JSON) to artifacts/telemetry/session_tokens.jsonl in the project directory. Each line is a complete JSON object:

{
  "timestamp": "2026-05-28T14:22:00Z",
  "agent": "Builder",
  "task": "TASK-003",
  "model": "claude-sonnet-4-6",
  "input_tokens": 4821,
  "output_tokens": 1203,
  "cache_read_tokens": 18420,
  "cache_write_tokens": 800,
  "turn_cost_usd": 0.02184
}

Updating pricing

When Anthropic or another provider changes pricing, update /home/user/codesleuth/config/pricing.json. The _last_updated field tracks when it was last changed. The tracker uses prefix matching for model families (e.g. "claude-sonnet" matches "claude-sonnet-4-6") with an Ollama fallback for unrecognized models.

Model-Adaptive Behavior

Every agent reads the active model from HANDOFF and adjusts its output format accordingly:

Model FamilyDetection PatternBehavior
Claude name contains claude XML-style structured output; extended reasoning for complex decisions; inject up to 20k tokens of Pattern Library and INTERFACES context
Kimi / Moonshot name contains kimi or moonshot Explicit ## Section headers throughout; inject full Pattern Library content (128k context handles it); include concrete output format examples per artifact
GPT / OpenAI name contains gpt, o1, or o3 Clean structured JSON where artifact output is requested; minimal prose; lead with structured data
Gemini name contains gemini Inject larger context windows aggressively (1M token support); include full codebase file lists and complete artifact content
Ollama / local default (all others) One step at a time; no more than 2 chained reasoning steps per response; explicit output format examples; context injections ≤ 8k tokens

Context Budget in HANDOFF

The Orchestrator maintains a context_budget object in HANDOFF that every agent reads before composing its prompt:

{
  "context_budget": {
    "total_tokens": 200000,
    "reserved_pipeline_state": 8000,
    "reserved_current_task": 40000,
    "available_history": 152000,
    "compression_threshold": 0.70,
    "compressed_tasks": []
  }
}
FieldPurpose
total_tokensModel context window size (from pricing.json)
reserved_pipeline_stateTokens reserved for state files (HANDOFF, CHECKPOINT, TASK-GRAPH summary)
reserved_current_taskTokens reserved for current task context (interfaces, schema, source files)
available_historyRemaining tokens available for conversation history
compression_thresholdWhen context hits this % (default 70%), rolling compression triggers
compressed_tasksList of task IDs whose full exchange has been compressed to one-line summaries
