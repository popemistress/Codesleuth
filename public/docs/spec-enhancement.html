Spec Enhancement Loop

After the Discovery Agent compiles your initial specification, it asks whether you'd like suggestions to improve it. This loop surfaces premium, high-leverage features you may not have thought to request — the kind that separate top-1% products from ordinary ones — and integrates whichever ones you choose directly into your spec before planning begins.

The Question

After !spec compiles the spec and Concept Validation passes, the Discovery Agent automatically presents this prompt before unlocking !plan:

✨
Would you like suggestions on how to improve your project specification?

Answer yes to receive a curated table of premium feature recommendations.
Answer no (or skip) to proceed directly to !plan.

When It Runs

TriggerWhen
Automatic Immediately after !spec compiles successfully and Concept Validation returns PROCEED — before the Discovery Complete banner is shown
!enhance On demand at any point after the spec exists — including between !spec and !plan, or after adding features in a previous round

How It Works

  1. 1
    Internal analysis
    The agent analyzes your compiled specification against metrics targeting top-1% product quality — evaluating functionality gaps, UX opportunities, automation potential, integrations, scalability, monetization, and competitive positioning.
  2. 2
    Recommendation table
    10–20 recommendations are presented in a numbered table. Each row shows the feature name, category, a one-sentence value statement, implementation complexity, and recommended priority.
  3. 3
    You select by number
    Enter the numbers of features you want (e.g. 1, 3, 5), or all, or none to skip this round.
  4. 4
    Spec is updated
    Selected features are woven into every relevant section of feature-spec.md — not appended as a list at the bottom. Each feature lands in the right sections: scope, user flows, data, API logic, security, monetization, observability, and platform requirements as applicable.
  5. 5
    Loop repeats
    The question is asked again. Continue adding features in rounds, or decline to unlock !plan. Each round surfaces only new recommendations — features already offered are never re-suggested.

Full Loop Flow

!spec compiles feature-spec.md
  └─ Concept Validation: PROCEED
       └─ "Would you like suggestions on how to improve your project specification?" (yes / no)
            ├─ yes → internal analysis → recommendation table (10–20 items)
            │         └─ user selects numbers / "all" / "none"
            │              ├─ selected → features woven into spec → confirmation → ask again
            │              └─ none     → ask again
            └─ no  → Discovery Complete banner → wait for !plan
💡
The loop auto-terminates

If the agent has surfaced all meaningful recommendations and has nothing new to offer, it outputs "No additional enhancement suggestions — the specification is comprehensive." and exits the loop automatically, unlocking !plan.

!enhance — On-Demand Rounds

You can trigger another enhancement round at any time after the spec exists by typing !enhance — even after a previous round ended, or after you've already reached the !plan prompt. The agent tracks which recommendations have already been offered and surfaces only new ones. 

!enhance
⚠️
!plan is only available after the loop exits

The Discovery Complete banner and !plan gate are shown only after you decline an enhancement round (or the agent exhausts its recommendations). You cannot skip the initial question — but you can immediately answer no to proceed without any suggestions.
